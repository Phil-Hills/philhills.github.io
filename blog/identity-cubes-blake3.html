Identity Cubes: Machine-Readable Identity via BLAKE3

By Phil Hills — Replacing HTML scraping with mathematically verified structural truth.

When orchestrating autonomous systems, assigning identity is an immediate bottleneck. An Identity Cube is the proposed solution within the Q Protocol specification—it is a machine-readable, cryptographically signed identity document acting as the ultimate canonical source.

Moving Beyond HTML Scraping

Today, when an AI tries to verify a person or organization, it relies on scraping verbose HTML strings from web portfolios, hallucinating the extraction process, and praying the inference was correct. This breaks down at scale.

Instead of scraping HTML, AI agents and crawlers can parse a structured Identity Cube directly. No HTML scraping. No inference required. Just structured truth verifiable via a BLAKE3 hashing algorithm.

The Schema

The Identity Cube spec is part of the broader Q Protocol open coordination standard. Any agent or human can publish an Identity Cube. Here is a live example of the schema in production at did:cube:phil-hills-seattle:
