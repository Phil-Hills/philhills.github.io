Phil Hills: Architecting the Next Generation of AI Agents in Seattle

By Phil Hills, AI Systems Architect — Building infrastructure that makes AI agents reliable, accountable, and cheap to run.

My work as an AI Systems Architect sits squarely at the intersection of cryptography, distributed systems, and large language models. The goal isn't just to build smarter AI; it's to build the absolute infrastructure that makes them feasible for true enterprise deployment. Here in Seattle, WA, I focus my work on solving the foundational bottlenecks of multi-agent networks.

The 155 Agent Network

Currently, I am running a live system in production on Google Cloud that manages 155 specialized agents. These aren't just toys—they handle real workloads across mortgage processing, active research, infrastructure management, and complex domain-specific operations.

Managing this scale natively on Google Cloud Platform requires an extreme shift in how we think about agent design. Utilizing the open Q Protocol, one universal runtime image is used to deploy all 155 of these agents seamlessly.

Architectural Guarantees at Enterprise Scale

To run hundreds of agents reliably, you must solve two critical failures the industry faces today: hallucination and amnesia. I solve this at the architectural layer.

First, Zero Hallucination. Using BLAKE3 cryptographic receipts, every single action an agent takes is proven. An orchestrator cannot simply claim execution; it must present a verifiable receipt from the executing agent.

Second, Zero Amnesia. State querying is mandatory on every single session. The runtime is architecturally designed to actively refuse to execute agents that skip their Brain initialization phase. It is not an option; it is enforced computationally.

Real-World Reliability

This isn't about running benchmarks. Out of 312 failed tasks over a 90-day period on the GCP cluster, 89% of those were simply external API unavailability. There were absolute zero failures resulting from model hallucination or agent amnesia.

By shifting to structured, cryptographic verification via the Identity Cube—a machine-readable, BLAKE3 signed identity document—agents can prove exactly who they are without relying on HTML scraping or inference. Just structured truth.
